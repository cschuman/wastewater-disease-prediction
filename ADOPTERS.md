## How to Add Yourself

### Option 1: Pull Request (Preferred)

1. Fork this repository
2. Edit this file (`ADOPTERS.md`)
3. Add your organization under the "Adopters" section using this template:

```markdown
### Your Organization Name

**Website:** https://your-website.com
**Use Case:** How you use the project (1-2 sentences)
**Contact:** (optional) GitHub handle or public email
**Since:** Month Year
```

4. Submit a pull request

### Option 2: GitHub Issue

Open an issue with:
- Subject: `[Adopter] Organization Name`
- Body: Website, use case, optional contact, date

We'll add you to the list.

### Option 3: Private Notification

If you can't publicly disclose usage, email the maintainers. We can:
- Count you in our internal metrics
- List you as "Anonymous Healthcare Organization" (with permission)
- Simply acknowledge your existence in aggregate stats

## Adopter Tiers

We recognize adopters at different levels:

| Tier | Description | Recognition |
|------|-------------|-------------|
| **User** | Using in any capacity | Listed in Adopters section |
| **Contributor** | Using + contributing back | Listed + contributor badge |
| **Sponsor** | Financial support | Listed + sponsor badge + logo on README |

---

## Requirements

To be listed, you must:
- Actually use the project (honor system)
- Agree to be listed publicly
- Provide accurate information

We reserve the right to remove listings that:
- Appear to be spam
- Misrepresent usage
- Violate our Code of Conduct
